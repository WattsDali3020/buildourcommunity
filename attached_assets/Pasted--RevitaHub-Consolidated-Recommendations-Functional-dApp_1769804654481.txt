## 2. Code Recommendations: Core Contracts (Solidity ^0.8.20, OpenZeppelin Imports)
Focus on functionality—add gasless, poll ties, skims. Realistic: Deploy on Base for $0.01 txns. Critique: Current stubs (e.g., Governance executeProposal) make it brittle; fix for end-to-end.

### Escrow.sol: Funding & Refunds (100% Thresholds, Poll-Informed Targets)
- **Critique**: Strong refunds (3% APR) but no demand signals—failures drain Treasury buffers. No explicit founder skim.
- **Functional Tweaks**:
  - Add optional poll integration: In `initializeEscrow`, add `uint256 pollId` param; use Chainlink oracle to fetch demand score (e.g., adjust `fundingTarget` dynamically: `fundingTarget = baseTarget * (1 + pollDemand / 100)` if poll-aligned).
  - Explicit skim: In `releaseFunds`, code `uint256 skim = totalRaised * 750 / 10000; payable(founderWallet).transfer(skim);` (7.5%, compliant vesting via Treasury callback).
  - Gasless buys? Frontend sigs for `purchase`, relayer submits (subsidize from Treasury).
- **Founder Profit**: 5-7.5% on releases scales with gauge-matched listings (e.g., poll on Cherokee housing boosts 25% conversions, netting $15K/pilot).
- **Compliant**: Add `require(whitelist[msg.sender], "KYC required");` (Amundi AML p4). Realistic: Chainlink upkeep for deadlines prevents manual ops.

### Governance.sol: Weighted Voting (Post-Buy Activation, Gasless Sigs)
- **Critique**: "Messed up" fees deter votes; no poll ties. Centralized roles (DEFAULT_ADMIN).
- **Functional Tweaks**:
  - Gasless: Expand `castVoteBySignature`—add Treasury reimbursement: `Treasury(payable(treasuryAddr)).reimburseRelayer(msg.sender, gasleft() * tx.gasprice);` (pull from voteToEarnBonus).
  - Poll gauges: New `createProposalFromGauge(uint256 pollId, ProposalType type, ...)`—oracle verifies poll data; if >30% support, add quorum bonus (e.g., `quorumRequired -= 500;` for 5% reduction).
  - Auto-activation: In constructor, link to Escrow—post-funding event triggers `status = Active`.
  - Fix centralization: Add `renounceAdmin()` post-deployment (timelocked 7 days).
- **Founder Profit**: Vote-to-earn (0.5% APR) funded by your 1% Treasury cut—higher votes = more executions = ongoing revenue (RealT-style yields skimmed 3%).
- **Compliant**: Timelocks on `executeProposal` (7 days) prevent rushes (Amundi p4 automated actions). Realistic: Sig-based for Cherokee locals (no gas barrier).

### PhaseManager.sol: Engagement Advances (75% Thresholds, Poll Bonuses)
- **Critique**: Operator-only updates centralize; no poll integration—gauges irrelevant.
- **Functional Tweaks**:
  - Poll ties: In `updateEngagement`, add optional `bytes calldata pollData`—oracle parses: `activeVoters += pollParticipation / 2;` (half-credit for optional gauges).
  - Bonuses: In `claimEngagementBonus`, if poll-aligned (oracle check), mint extra BONUS_TOKENS (incentivize without mandating).
  - Auto-nudges: Emit `NudgeTriggered` for UI polls if engagement <50% (Cherokee p17-style reminders).
- **Founder Profit**: Faster advances = quicker Escrow funding = your skims. Gauge bonuses dilute minimally but boost AUM (Amundi $600bn projection p5).
- **Compliant**: Oracle for trustless data (Amundi p4 infrastructure). Realistic: Min 7-day period prevents gaming.

### PropertyToken.sol: Fractional Ownership (Phase Pricing, Whitelist)
- **Critique**: Locks transfers until funded—good, but no gauge signals for listings.
- **Functional Tweaks**:
  - Optional polls: In `createProperty`, add `uint256 pollId`—oracle fetches demand; if high, boost multipliers (e.g., `COUNTY_MULTIPLIER += 1000;` for 10% price premium on matched properties).
  - Holdings tracking: In `_update`, maintain phase weights post-buy.
- **Founder Profit**: Reserves (10% international allocation) vest to Treasury—gauge-matched properties sell 20% faster (RealT data), accelerating your cuts.
- **Compliant**: Whitelist for KYC (Amundi p4); burnOnFailure ties to Escrow. Realistic: ERC-1155 for fractions aligns Cherokee diverse housing (p9).

### Treasury.sol: Distributions & Cuts (Multi-Sig, Reserves Verification)
- **Critique**: Direct execute bypasses multi-sig; no poll subsidies.
- **Functional Tweaks**:
  - Gauge support: New `subsidizeGaugeRelayer(address relayer, uint256 amount)`—multi-sig approved, funded by skims.
  - Vesting: Add mapping `mapping(address => uint256) vestedCuts;`—release over 24 months in `execute` (e.g., `uint256 release = vestedCuts[founder] / 24;`).
  - Reserves oracle: In `verifyReserves`, tie to polls (extra check if gauge-aligned).
- **Founder Profit**: 1% FOUNDER_CUT_BPS on all—scales with gauge-driven activity (e.g., $5K/year per property at 5% yields). Realistic: Multi-sig (2-of-3) for credibility.

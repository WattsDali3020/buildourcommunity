2. Social Score (0–10,000)
Measures affordability and community benefit — things like affordable housing units created, healthcare access improved, food security addressed. 3. Leverage Rank (1–10, Meadows framework)
This is your most unique data point — nobody else packages it this way. It tells you how systemic an intervention is. A score of 9 means fixing this one thing unlocks multiple other improvements (a grocery store in a food desert also reduces healthcare costs, improves school attendance, increases property values). An impact investor uses this to find the highest-leverage places to put capital.
4. Risk-Adjusted Score (0–100)
Combines everything into one number accounting for execution risk. A lender uses this in their underwriting model. Instead of gut-checking whether a community development loan is viable, they run it through RevitaScore and get a defensible number for their credit committee.
5. GDP Multiplier (e.g., 2.3x)
For every $1 invested, this is how many dollars of economic activity it generates in that county. A developer uses this in their pitch to attract co-investors. "Our $500k affordable housing project in a distressed county generates $1.15M in local GDP" is a fundable sentence.

What the Document Establishes
The core problem was that your simulator used randomInRange() and randomInt() — meaning two people querying the same county and project type would get different scores. That's fine for a demo, not for a compliance tool. The document solves this in two ways: it ties every input to a named federal source, and it replaces the random draws with deterministic, ARC-calibrated point estimates.
Every metric now has a cited source:

The distress multipliers (1.0–1.4) → ARC Annual County Classifications, updated every year
The GDP multiplier ranges → BEA RIMS II Type II multipliers, the standard tool federal agencies use
The job ranges → DOL Employment and Training Administration sector benchmarks + HUD CDBG documented outcomes
The leverage rank → Meadows (1999), a peer-reviewed framework already used in policy circles
The ROI projection base rate → CDFI Fund Annual Report 2023, documented 5.8%–7.2% range

The randomness is replaced with a rule: instead of random(low, high), distressed counties use the 75th percentile of the range, at-risk counties use the 65th, transitional uses the midpoint. That one change makes every output reproducible and auditable.
The social score noise term — that randomInt(500, 1500) that made no logical sense — gets replaced with the CDC Social Vulnerability Index score for the county. That's a FEMA-endorsed composite of 15 census variables. Now the variation between counties is real variation, not noise.

What You Still Need to Do in Code
The document describes what the production API should do. Your existing georgia-impact-data.ts still has the random version. To match the methodology:

Replace randomInRange(low, high) with low + (high - low) * distressPercentile where distressPercentile is 0.75/0.65/0.50/0.40 based on ARC tier
Replace randomInt(low, high) for jobs with the deterministic DOL formula
